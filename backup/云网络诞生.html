1. 产业竞争核心

  • 芯片

2. 转发能力发展路线

虚拟交换机发展从10Gbit/s（以原始openvswitch为代表的内核态转发，使用 kernel 作为 datapath） -> 25Gbit/s（以dpdk技术为依托的内核态转发，使用用户空间作为 datapath）-> 100Gbit/s（硬件卸载）

3. 云+管+边+端

”云“指提供中心计算的公有云，“管“则是指有线/无线通讯方式，“边”是指边缘计算，“端”则涵盖了智能传感、智能终端和智能设备。

云端：将需要大算力的功能（如大数据、大模型、L7 service等）放在云端，由云端高性能、可弹性扩缩的云服务器进行处理

边缘计算：大量在物联网行业应用，其作为物联网边缘的“小脑”，在靠近物或数据源头的边缘侧就近提供计算和智能服务，解决了把计算放在云端而引起的网络延时、网络拥塞和服务质量下降等问题，也极大缓解了终端的算力压力，为物联网的进一步发展提供了可能性。

终端：利用终端芯片本地实时处理对即时性要求很高的内容，如手机拍照的实时美化，3D游戏的画面渲染等。

4. Vxlan技术对云网络的意义

  1. 避免了对物理网络设备（交换机、路由器）的升级
  2. 大大节省了物理设备路由表地址空间
  3. 提供了逻辑意义上的租户隔离

5. vxlan技术对云网络的要求

  1. 提供租户层面的虚拟设备，如虚拟交换机、虚拟路由器、虚拟防火墙、虚拟负载均衡器等
  2. 共享物理网络对保障每个租户间的QoS、限速和调度能力提出了更高的要求。

6. 管控的变革

人机接口CLI（Command Line） -> 机机接口API（Application Programming Interface）的转变，实现可编程的中央控制器来集中管理。
