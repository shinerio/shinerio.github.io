# 1. 全动态BGP（多线EIP）
云服务提供商的公网IP地址通过BGP与多个运营线直连的链路播报给多个运营商。BGP类型的带宽具备动态路由收敛能力，可靠性和抗DDos能力好，但价格相对静态BGP来说较贵。云服务提供商可以根据设定的寻路协议实时自动优化网络结构，以保持客户使用的网络持续稳定、高效。
# 2. 静态BGP（单线静态EIP）
静态BGP只与一家运营商线路直连，不具备动态路由收敛能力，可靠性低，无抗DDos能力，价格较低。一般网络结构发生变化时，无法实时自动调整网络设置以保障用户体验。
# 3. 优选BGP（精品EIP）
是特定方向的优质线路。使用BGP协议与多家主流运营商线路互联对接，建立直连中国内地的公网互联路径，提供中国-香港区域与中国内地间的低时延、高质量的网络互通。适用于部署在香港，面向大陆提供优质服务的客户。
# 4. AnycastEip
同一个EIP可以在全球多个POP**同时接入**云厂商网络，将客户端的访问请求通过云厂商骨干网分发至最佳终端节点，减少公网传输时延。适用于后端业务部署在特定区域，但需要面向全球提供高可靠、低时延的服务。
# 5. GlobalEip
GEIP主要是用于同一个城域内通过云内骨干网络进行加速。以华南和华东为例，华南和华东属于两个不同的城域，比如华东-上海一和华东-上海二，均属于华东城域内。GEIP提供**指定POP点接入**，如可以将GEIP在苏州的POP点接入，通过华为云骨干网将流量路由到上海一的后端。适用于将指定区域client的流量快速接入到server所在区域。
