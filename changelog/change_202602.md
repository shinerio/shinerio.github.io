## fix: 修复无序列表文章预览只显示"-"的问题
date: 2026-02-24
- 修复 `SiteGenerator.ts` 中 `extractDescription` 方法的 bug
- 原正则 `/\[.*?\]\(.*?\)/g` 将 Markdown 链接整体删除（包括链接文字），导致以 `[text](url)` 格式组成的无序列表条目在提取描述时仅剩 `-`，首页预览只显示一个"-"
- 将替换规则改为 `/\[([^\]]*)\]\(.*?\)/g` 替换为 `$1`，保留链接文字，丢弃 URL 部分，使描述内容正确显示

## fix: 修复含#号的URL链接渲染失败问题
date: 2026-02-23
- 修复 `MetadataParser.ts` 中 Obsidian 标签处理正则表达式的 bug
- 原正则 `/#([a-zA-Z0-9...]+)/g` 会错误匹配 URL 中的锚点片段（如 `https://example.com/page#readme` 中的 `#readme`），导致含 `#` 的 Markdown 链接渲染失败
- 新增负向后瞻 `(?<!\S)` 限制，确保 `#` 只在行首或空白字符后才会被识别为 Obsidian 标签
