# Presenter TOC Design

**Date:** 2026-03-15
**Scope:** Add a hover-revealed table of contents to presenter mode on article detail pages, reusing the normal article TOC instead of generating a separate presenter-only structure.

---

## Goals

- Presenter mode should expose the same heading directory as the normal article page.
- The TOC should stay hidden by default and slide in from the right edge on hover.
- TOC links should scroll the presenter container, not the underlying page.
- Active section highlighting should continue to work inside presenter mode.
- The change must not alter normal article-page TOC behavior.

---

## Chosen Approach

Reuse the existing article TOC markup from `templates/article.html` and clone it into presenter mode when the presenter shell is created.

This avoids duplicating heading parsing rules, nesting, labels, and link targets. Presenter mode already clones the article content into an isolated shell, so the TOC can follow the same pattern: copy the rendered TOC DOM, attach presenter-specific event handlers, and style it as a right-side drawer.

---

## Alternatives Considered

### 1. Rebuild TOC in presenter mode from headings

- Pros: fully self-contained presenter shell
- Cons: duplicates heading extraction logic already handled during site generation
- Rejected because it creates drift risk between normal page and presenter mode

### 2. Keep TOC permanently visible in presenter mode

- Pros: simplest implementation
- Cons: does not match the requested hidden-by-default interaction and permanently consumes horizontal space
- Rejected because it misses the product requirement

---

## Interaction Design

- Presenter shell gets a fixed, narrow hover trigger along the right edge.
- The trigger is transparent but captures hover/focus.
- Hovering the trigger reveals a right-side TOC drawer with the existing TOC card content.
- Hovering inside the drawer keeps it open.
- Leaving both the trigger and drawer collapses it.
- Clicking a TOC entry scrolls `state.shellInner` so the matching heading is aligned near the top of the presenter viewport.
- The current TOC item receives the same `.active` class used on the normal page.

---

## DOM and State Changes

### `templates/article.html`

- Add a stable hook to the existing TOC sidebar so presenter mode can find and clone it safely.
- Keep the normal article-page structure unchanged for existing styles and scripts.

### `templates/assets/js/article-presenter.js`

- Detect whether the source page has a TOC sidebar.
- When entering presenter mode:
  - clone the TOC sidebar into a presenter-specific drawer
  - add a right-edge hover trigger
  - bind drawer hover/open/close behavior
  - bind TOC link clicks to presenter scrolling
  - compute active heading state against presenter content and presenter scroll position
- Skip all TOC-specific work if the article has no TOC.

### `templates/assets/css/style-addition.css`

- Add presenter-only styles for:
  - right-edge trigger zone
  - hidden drawer state
  - revealed drawer state
  - TOC card sizing, scrolling, and pointer behavior
- Preserve existing normal-page TOC styles in `templates/assets/css/style.css`.

---

## Scrolling and Active Link Behavior

- Existing article TOC links target heading IDs already generated by `SiteGenerator`.
- Because presenter mode clones the article DOM, the same IDs exist inside the presenter article.
- Presenter TOC click handling should:
  - prevent default navigation
  - find the matching heading inside `state.presenterArticle`
  - compute offset relative to `state.stage`
  - scroll `state.shellInner` with smooth behavior
- Active-link updates should use presenter heading positions, not the source page positions.

---

## Constraints

- No new dependencies.
- No mobile presenter support changes; the existing coarse-pointer guard stays in place.
- No change to TOC generation rules (`h1`/`h2`/`h3` only).
- No new presenter mode controls outside the TOC drawer.

---

## Testing Strategy

### Automated

- Add a site-generation test that verifies article output contains the stable TOC hook needed by presenter mode.
- Add a focused presenter script test for the DOM behavior if the current Jest environment can support it with lightweight stubs.
- At minimum, verify presenter initialization only creates TOC drawer affordances when the source article contains a TOC.

### Manual

- Open an article with headings.
- Enter presenter mode.
- Move the cursor to the right edge and confirm the TOC slides in.
- Move into the drawer and confirm it stays open.
- Move away and confirm it closes.
- Click TOC links and confirm the presenter scroll container jumps to the correct heading.
- Scroll presenter content and confirm active TOC highlighting updates.
